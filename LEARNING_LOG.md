# 翱翔之翼官网平台学习记录

这个文档会记录我们每次对网站做了什么、为什么这样做，以及你作为网页开发初学者需要理解的知识点。后续每次调整功能，我都会同步更新这里。

## 2026-05-31：确定项目方向并创建首版工程

### 本次目标

把“阜阳师范大学中科协‘翱翔之翼’大学生志愿服务队官网”从普通展示页升级为一个可以长期更新的平台，包含官网展示、志愿者管理、活动报名、证书生成、课程学习、器材教具、学生成果和竞赛成果等模块。

### 我做了什么

1. 查看当前项目目录，确认它是空目录，还不是 Git 仓库。
2. 用 Vite 创建了一个 React 项目。
3. 安装了项目依赖。
4. 从你提供的真实活动照片中复制了一部分到 `public/media`，作为网站首版视觉素材。
5. 修改了网页标题、语言和 SEO 描述，让网站未来更适合被搜索引擎识别。
6. 创建了这份 `LEARNING_LOG.md` 学习记录文档。

### 这些操作是什么意思

`Vite` 是一个前端项目构建工具。你可以把它理解为“网页开发启动器”：它帮我们快速创建项目、启动本地预览服务器、打包上线文件。

`React` 是一个前端框架。它适合做这种有很多栏目、很多交互、后续还要持续增加功能的网站平台。

`public/media` 是静态资源目录。放在这里的图片可以直接被网页访问，例如 `/media/sxx-01.jpeg`。

`index.html` 是网站最基础的入口文件。里面的 `<title>` 会显示在浏览器标签页，也会影响搜索结果标题；`description` 会帮助搜索引擎理解网站内容。

### 为什么这样做

如果只写一个普通 HTML 页面，后续你要加“报名系统”“证书生成”“志愿者管理”会很难维护。React 项目更适合把不同功能拆成模块，后面可以逐步接数据库、后台接口和登录权限。

### 你需要先理解的概念

- 前端：用户在浏览器里看到和操作的页面。
- 后端：负责保存数据、处理报名、登录、证书生成等逻辑的服务器程序。
- 数据库：保存志愿者、活动、报名、课程、器材等信息的地方。
- 静态资源：图片、视频、图标、PDF 等文件。
- 本地开发服务器：只在你电脑上运行的预览环境，用来开发和调试。

### 我做了什么

3. 简化了 `src/index.css`，保留全局字体、背景、滚动等基础样式。
4. 做了这些栏目：
   - 翱翔之翼信息介绍栏
   - 志愿者信息管理栏
   - 活动展示栏
   - 志愿者招新栏

   - 少儿编程学习栏
   - 器材教具栏
5. 活动报名现在可以填写姓名、学院/专业、联系方式、活动和岗位。
6. 报名数据会暂存在浏览器的 `localStorage`。
7. 报名后，志愿者管理表格会出现记录，并可以点击“生成证书”。

### 这些代码是什么意思

`App.jsx` 是 React 组件文件。你可以把一个组件理解为“页面上的一块功能区域”。现在为了让你更容易学习，我先把所有模块放在一个文件里，后续功能多了再拆成多个组件文件。

`useState` 是 React 用来保存页面状态的工具。比如报名表里输入的姓名、电话，以及报名记录列表，都属于状态。

`localStorage` 是浏览器自带的小型本地存储。它适合做学习阶段的演示：刷新页面后，报名记录仍然能保留。但它不是正式数据库，正式上线后要换成后端数据库。

`map` 是 JavaScript 里常见的数组方法。页面里的导航、活动卡片、课程列表、器材列表，都是通过数组 `map` 自动生成的。这样以后加一个活动，只需要往数组里加一条数据，不需要复制一大段 HTML。

### 为什么这样做

你希望网站以后可以不停更新新功能，所以首版不能只写死一张页面。我把内容先做成“数据数组 + 页面循环渲染”的形式，这是平台开发的基础思路：数据变了，页面自动跟着变。

报名和证书先做前端演示，是为了让你尽快看到完整流程：

```text
填写报名表 -> 进入志愿者管理表 -> 点击生成证书
```

后续我们再把这条流程升级成：

```text
填写报名表 -> 后端保存到数据库 -> 管理员审核 -> 签到签退 -> 记录时长 -> 生成可核验证书
```

### 你需要理解的新概念

- 组件：页面中可复用的一块功能。
- 状态：页面会变化的数据，例如报名表内容。
- 表单：用户输入信息的地方。
- 本地存储：浏览器临时保存数据的地方。
- 数据驱动页面：先整理数据，再用代码把数据渲染成页面。

### 当前限制

1. 现在还没有真正的登录系统。
2. 现在还没有后端和数据库。
3. 证书只是网页弹窗演示，还不能导出 PDF。
4. 管理员审核、服务时长、证书编号核验还没有真实业务规则。

这些限制不是问题，首版的任务是先把功能流程跑通。后面每次加功能，我们会逐步把“演示版”替换成“正式版”。

### 为什么这样做

这个网站未来要靠搜索进入，陌生人打开后通常只会先看几秒钟。首屏负责吸引和建立信任，第二屏负责证明项目真的做过事情、有成果、有报道。后面的报名、管理、课程和器材模块再承担具体功能。
## 2026-05-31：补充修正页面滚动与科技背景

### 我做了什么

1. 把页面外层的 `overflow: hidden` 改成了 `overflow-x: hidden`。
2. 把科技背景里的圆形光晕改成更简洁的线性光感和网格线。

### 这是什么意思

`overflow` 控制页面内容超出容器后怎么显示。`overflow: hidden` 会把超出部分藏起来，如果放在最外层，页面可能就不能继续往下滚动。现在改成 `overflow-x: hidden`，意思是只隐藏横向多出来的部分，纵向内容仍然可以滚动。

科技感不只是“加一个蓝色光圈”。更适合官网的做法，是用稳定的深色底、网格线、光线层次和真实活动照片来建立专业感。

我为什么主要改 CSS：

- 这次你的需求是视觉风格统一，所以优先改 CSS，可以一次影响多个栏目，避免每个栏目重复写一遍样式。

你可以这样理解：
- `#overview`、`#volunteers`、`#activities` 这类以 `#` 开头的是页面栏目 ID，用来控制某一个具体栏目。
- `.content-section`、`.split-section`、`.signup-section` 这类以 `.` 开头的是通用样式类，用来控制一组页面的共同外观。
- `background-image` 同时叠了三层：深色遮罩、科技网格、真实活动照片。这样照片不会抢文字，页面也更有冲击力。

### 这些概念是什么意思

前端：你在浏览器里看到的页面，比如首页、报名表、志愿者管理栏。

后端：运行在电脑或服务器上的程序，负责接收前端请求、处理数据、保存数据。这里的入口是 `server/index.js`。

接口：前端和后端约定好的“通信地址”。比如前端提交报名表时，会请求 `POST /api/registrations`。

数据库：长期保存数据的地方。现在我们用 SQLite，它会在项目里的 `data/aoxiang.sqlite` 生成一个数据库文件。

数据表：数据库里的分类存储区域。现在有 `registrations` 报名表、`activities` 活动表、`accounts` 账号表。

### 为什么先用 SQLite

SQLite 不需要额外安装大型数据库软件，Node 现在可以直接使用它。对学习来说，它能让你先理解“数据怎么保存、接口怎么调用、前后端怎么连接”，不用一开始就被数据库服务器配置卡住。

以后正式部署到学院服务器或云服务器时，可以继续用 SQLite 做小规模运行；如果报名、文章、文件、账号越来越多，再迁移到 MySQL 或 PostgreSQL。

## 2026-06-02：如何验证后端是否真的接上

这次验证分成两步：

1. 先验证后端接口：访问 `http://127.0.0.1:3001/api/health`。如果返回 `ok: true` 和 `database.type: SQLite`，说明后端服务和数据库文件已经正常工作。
